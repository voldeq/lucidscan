# Swift

**Support tier: Full**

Swift projects have dedicated tools across linting, formatting, type checking, testing, coverage, security, and duplication detection.

## Detection

| Method | Indicators |
|--------|-----------|
| **File extensions** | `.swift` |
| **Marker files** | `Package.swift` |
| **Version detection** | `swift-tools-version` from `Package.swift`, `.swift-version` |

## Tools by Domain

| Domain | Tool | Notes |
|--------|------|-------|
| **Linting** | SwiftLint | Style and convention enforcement; 200+ rules |
| **Formatting** | SwiftFormat | Opinionated code formatter by Nick Lockwood |
| **Type Checking** | Swift compiler | Compiler diagnostics via `swift build` |
| **Testing** | swift test | XCTest via Swift Package Manager |
| **Coverage** | llvm-cov | Coverage data from `swift test --enable-code-coverage` |
| **Security (SAST)** | OpenGrep | Swift-specific vulnerability rules |
| **Security (SCA)** | Trivy | Scans Package.resolved |
| **Duplication** | Duplo | Scans `.swift` files |

## Linting

[SwiftLint](https://github.com/realm/SwiftLint) enforces Swift style and conventions with 200+ configurable rules.

### Installation

```bash
# macOS (Homebrew)
brew install swiftlint

# Linux
# See https://github.com/realm/SwiftLint#installation
```

### Severity Mapping

| Severity | Rule Categories |
|----------|----------------|
| **HIGH** | `force_cast`, `force_try`, `force_unwrapping`, `implicitly_unwrapped_optional`, `unused_capture_list` |
| **MEDIUM** | `cyclomatic_complexity`, `function_body_length`, `file_length`, `type_body_length`, `nesting`, `empty_count`, `first_where` |
| **LOW** | `line_length`, `trailing_whitespace`, `identifier_name`, `type_name`, `todo`, `unused_import`, `syntactic_sugar` |

### Configuration

SwiftLint can be configured via `.swiftlint.yml` in the project root. LucidShark uses the project's existing configuration if present.

## Formatting

[SwiftFormat](https://github.com/nicklockwood/SwiftFormat) is an opinionated code formatter for Swift.

### Installation

```bash
# macOS (Homebrew)
brew install swiftformat

# Linux
# See https://github.com/nicklockwood/SwiftFormat#installation
```

SwiftFormat checks formatting with `--lint` mode and applies fixes directly.

## Type Checking

The Swift compiler provides type checking via `swift build`. All compiler errors and warnings are reported as type checking issues.

### Severity Mapping

| Level | Severity |
|-------|----------|
| `error` | HIGH |
| `warning` | MEDIUM |
| `note` | LOW |

### Requirements

- Swift toolchain installed (Xcode on macOS, or swift.org toolchain)
- `Package.swift` in the project root

## Testing

Tests run via `swift test`, which executes XCTest-based tests in the `Tests/` directory.

### Features

- Parses individual test pass/fail results
- Extracts failure messages with file and line info
- When coverage domain is enabled, adds `--enable-code-coverage` flag

### Requirements

- `Package.swift` with test targets defined
- Test files in `Tests/` directory using XCTest framework

## Coverage

Coverage data is generated by `swift test --enable-code-coverage` and parsed from the llvm-cov JSON export format.

### How It Works

1. The Swift test runner adds `--enable-code-coverage` when coverage domain is enabled
2. The coverage plugin uses `swift test --show-codecov-path` to find the JSON report
3. Per-file and per-line coverage data is extracted from the llvm-cov export

### Severity Escalation

| Coverage | Severity |
|----------|----------|
| < 50% | HIGH |
| < threshold - 10% | MEDIUM |
| < threshold | LOW |

## Security

OpenGrep includes SAST rules for Swift code covering common vulnerabilities. Trivy scans `Package.resolved` for known vulnerabilities in Swift package dependencies.

## Duplication

Duplo scans `.swift` files for duplicate code blocks across the project.

## Timeouts

| Tool | Timeout |
|------|---------|

| SwiftFormat | 120s |
| swift build | 300s |

## Example Configuration

```yaml
version: 1
project:
  languages: [swift]
pipeline:
  linting:
    enabled: true
    tools: [{ name: swiftlint }]
  formatting:
    enabled: true
    tools: [{ name: swiftformat }]
  type_checking:
    enabled: true
    tools: [{ name: swift_compiler }]
  testing:
    enabled: true
    tools: [{ name: swift_test }]
  coverage:
    enabled: true
    threshold: 80
    tools: [{ name: swift_coverage }]
  security:
    enabled: true
    tools: [{ name: opengrep }, { name: trivy }]
  duplication:
    enabled: true
    tools: [{ name: duplo }]
```

## Known Limitations

- Swift toolchain is primarily available on macOS and Linux (not Windows)
- Coverage requires `swift test --enable-code-coverage` flag (enabled automatically when coverage domain is active)
- SwiftLint and SwiftFormat must be installed separately (not managed by LucidShark)
