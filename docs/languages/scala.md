# Scala

**Support tier: Full**

Scala projects have dedicated tools across linting, formatting, type checking, testing, coverage, security, and duplication detection.

## Detection

| Method | Indicators |
|--------|-----------|
| **File extensions** | `.scala`, `.sc` |
| **Marker files** | `build.sbt` |
| **Version detection** | `scalaVersion` in `build.sbt`, `.scala-version` file |

## Tools by Domain

| Domain | Tool | Notes |
|--------|------|-------|
| **Linting** | [Scalafix](#linting) | Refactoring and linting tool for Scala |
| **Formatting** | [Scalafmt](#formatting) | Standard Scala code formatter |
| **Type Checking** | [Scala Compiler](#type-checking) | Via `sbt compile`, `mvn compile`, or `gradle compileScala` |
| **Testing** | [sbt test](#testing) | Via sbt, Maven, or Gradle with JUnit XML report parsing |
| **Coverage** | [Scoverage](#coverage) | Scala code coverage via sbt-scoverage, Maven, or Gradle plugins |
| **Security (SAST)** | OpenGrep | Scala-specific vulnerability rules |
| **Security (SCA)** | Trivy | Dependency scanning via build files |
| **Duplication** | Duplo | `.scala` and `.sc` file support |

## Build System Support

Scala plugins auto-detect the project's build system:

1. **sbt** -- detected via `build.sbt` marker file (preferred)
2. **Maven** -- detected via `mvnw` wrapper or `pom.xml` with scala-maven-plugin
3. **Gradle** -- detected via `gradlew` wrapper or `build.gradle` with Scala plugin

## Linting

**Tool: [Scalafix](https://scalacenter.github.io/scalafix/)**

Scalafix is a refactoring and linting tool for Scala. It supports built-in rules (e.g., `RemoveUnused`, `DisableSyntax`, `OrganizeImports`) and custom rules.

**Installation:**
```bash
# Via Coursier
cs install scalafix

# Or add sbt-scalafix plugin to project/plugins.sbt:
addSbtPlugin("ch.epfl.scala" % "sbt-scalafix" % "0.11.1")
```

**Configuration:** `.scalafix.conf` in project root.

```conf
rules = [
  RemoveUnused
  DisableSyntax
  OrganizeImports
]

DisableSyntax.noVars = true
DisableSyntax.noNulls = true
```

**Auto-fix:** Scalafix supports automatic refactoring. LucidShark runs `scalafix --check` for detection and `scalafix` for fixes.

## Formatting

**Tool: [Scalafmt](https://scalameta.org/scalafmt/)**

Scalafmt is the standard code formatter for Scala, supporting Scala 2 and Scala 3.

**Installation:**
```bash
# Via Coursier
cs install scalafmt

# Or add sbt-scalafmt plugin to project/plugins.sbt:
addSbtPlugin("org.scalameta" % "sbt-scalafmt" % "2.5.2")
```

**Configuration:** `.scalafmt.conf` in project root.

```conf
version = 3.7.17
runner.dialect = scala3
maxColumn = 100
```

**Auto-fix:** Scalafmt always supports auto-fix (formatting is inherently a fix operation).

## Type Checking

**Tool: Scala Compiler (scalac)**

LucidShark runs the Scala compiler via the detected build tool to extract type errors and warnings:

- `sbt compile` for sbt projects
- `mvn compile` for Maven projects
- `gradle compileScala` for Gradle projects

Compiler output is parsed to extract errors (HIGH severity) and warnings (MEDIUM severity) with file paths, line numbers, and column positions.

## Testing

**Tool: sbt test / Maven / Gradle**

LucidShark runs tests via the detected build tool and parses JUnit XML reports generated by test frameworks like ScalaTest, specs2, MUnit, and others.

- **sbt**: Runs `sbt test`, parses `target/test-reports/` or `target/scala-X.Y/test-reports/`
- **Maven**: Runs `mvn test`, parses `target/surefire-reports/`
- **Gradle**: Runs `gradle test`, parses `build/test-results/test/`

All major Scala test frameworks (ScalaTest, specs2, MUnit, ScalaCheck) generate JUnit XML reports that LucidShark can parse.

## Coverage

**Tool: [Scoverage](https://github.com/scoverage/scalac-scoverage-plugin)**

Scoverage instruments Scala code at the statement level for precise coverage measurement.

**Setup for sbt:**
```scala
// project/plugins.sbt
addSbtPlugin("org.scoverage" % "sbt-scoverage" % "2.0.11")
```

Run with:
```bash
sbt coverage test coverageReport
```

**Setup for Maven:**
```xml
<plugin>
    <groupId>org.scoverage</groupId>
    <artifactId>scoverage-maven-plugin</artifactId>
    <version>2.0.2</version>
</plugin>
```

LucidShark parses the Scoverage XML report (`scoverage.xml`) with per-file statement coverage, including threshold violation detection.

## Security

**SAST:** OpenGrep includes SAST rules for Scala code covering common vulnerabilities (injection, insecure crypto, etc.).

**SCA:** Trivy scans `build.sbt`, `pom.xml`, and `build.gradle` dependency definitions for known vulnerabilities.

## Duplication

Duplo supports `.scala` and `.sc` files for cross-file duplicate code detection.

## Example Configuration

```yaml
version: 1
project:
  languages: [scala]
pipeline:
  linting:
    enabled: true
    tools: [{ name: scalafix }]
  formatting:
    enabled: true
    tools: [{ name: scalafmt }]
  type_checking:
    enabled: true
    tools: [{ name: scala_compile }]
  security:
    enabled: true
    tools:
      - { name: trivy, domains: [sca] }
      - { name: opengrep, domains: [sast] }
  testing:
    enabled: true
    tools: [{ name: sbt }]
  coverage:
    enabled: true
    tools: [{ name: scoverage }]
    threshold: 80
  duplication:
    enabled: true
    threshold: 5.0
```
