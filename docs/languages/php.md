# PHP

**Support tier: Full**

PHP has dedicated tool coverage across all quality domains including linting, formatting, type checking, testing, coverage, duplication, and security scanning.

## Detection

| Method | Indicators |
|--------|-----------|
| **File extensions** | `.php` |
| **Marker files** | `composer.json` |
| **Version detection** | `require.php` from `composer.json`, `.php-version` file |

## Tools by Domain

| Domain | Tool | Auto-Fix | Notes |
|--------|------|----------|-------|
| **Linting** | PHP_CodeSniffer (phpcs) | Yes (phpcbf) | Configurable coding standards (PSR-12, PSR-1, custom) |
| **Formatting** | PHP-CS-Fixer | Yes | Coding standards fixer, extensive rule library |
| **Type Checking** | PHPStan | -- | Levels 0-9, strict mode available |
| **Security (SAST)** | OpenGrep | -- | PHP-specific vulnerability rules |
| **Security (SCA)** | Trivy | -- | Scans `composer.lock` |
| **Testing** | PHPUnit | -- | JUnit XML report parsing, coverage integration |
| **Coverage** | PHPUnit (Clover XML) | -- | Requires Xdebug or PCOV extension |
| **Duplication** | Duplo | -- | Configurable min lines and threshold |

## Linting

**Tool: [PHP_CodeSniffer](https://github.com/squizlabs/PHP_CodeSniffer)**

PHP_CodeSniffer tokenizes PHP files and detects violations of a defined set of coding standards. LucidShark uses the `phpcs` binary for linting and `phpcbf` for auto-fixing.

- Supports auto-fix via `phpcbf` (companion tool)
- Multiple coding standards: PSR-12, PSR-1, Squiz, PEAR, custom rulesets
- Configurable via `phpcs.xml`, `phpcs.xml.dist`, or `.phpcs.xml`
- JSON output format for structured issue parsing

```yaml
pipeline:
  linting:
    enabled: true
    tools:
      - name: phpcs
```

## Formatting

**Tool: [PHP-CS-Fixer](https://cs.symfony.com/)**

PHP Coding Standards Fixer automatically fixes code to follow standards. Supports a wide range of formatting rules.

- Supports auto-fix via `php-cs-fixer fix`
- Check-only mode via `--dry-run`
- Configurable via `.php-cs-fixer.php` or `.php-cs-fixer.dist.php`

```yaml
pipeline:
  formatting:
    enabled: true
    tools:
      - name: php_cs_fixer
```

## Type Checking

**Tool: [PHPStan](https://phpstan.org/)**

PHPStan finds bugs in code without running it. It supports analysis levels 0-9, where higher levels enable stricter checks.

- Levels 0-9 (configurable, default uses project config)
- JSON error output format
- Configurable via `phpstan.neon`, `phpstan.neon.dist`, or `phpstan.dist.neon`

```yaml
pipeline:
  type_checking:
    enabled: true
    tools:
      - name: phpstan
```

## Testing

**Tool: [PHPUnit](https://phpunit.de/)**

The standard PHP testing framework.

- JUnit XML report parsing for structured test results
- Automatic coverage instrumentation when coverage domain is enabled (`--coverage-clover`)
- Failure and error extraction with file/line information

```yaml
pipeline:
  testing:
    enabled: true
    tools:
      - name: phpunit
```

## Coverage

**Tool: [PHPUnit Coverage (Clover XML)](https://docs.phpunit.de/en/11.5/code-coverage.html)**

Parses Clover XML coverage data generated by PHPUnit. Requires the Xdebug or PCOV PHP extension for coverage collection.

- Clover XML format parsing with per-file coverage tracking
- Missing line number extraction
- Threshold-based pass/fail
- Looks for coverage data in: `coverage-clover.xml`, `coverage.xml`, `build/logs/clover.xml`, `clover.xml`

```yaml
pipeline:
  coverage:
    enabled: true
    tools: [{ name: phpunit_coverage }]
    threshold: 80
```

## Security

Security tools (OpenGrep, Trivy) are language-agnostic. See the domain-specific sections in the [main documentation](../main.md) for details.

OpenGrep includes SAST rules for PHP code, covering common web vulnerability patterns like SQL injection, XSS, and file inclusion.

Trivy SCA scans PHP manifests: `composer.lock`.

## Duplication

Duplo scans `.php` files for duplicate code blocks. Configure the minimum lines and duplication threshold:

```yaml
pipeline:
  duplication:
    enabled: true
    threshold: 5.0
```

## Prerequisites

PHP tools are installed via Composer. Ensure these dev dependencies are present:

```bash
composer require --dev squizlabs/php_codesniffer
composer require --dev phpstan/phpstan
composer require --dev phpunit/phpunit
composer require --dev friendsofphp/php-cs-fixer
```

For coverage collection, install either Xdebug or PCOV:

```bash
pecl install xdebug
# or
pecl install pcov
```

## Example Configurations

### Production (all domains, strict checks)

```yaml
version: 1
project:
  languages: [php]
pipeline:
  linting:
    enabled: true
    tools: [{ name: phpcs }]
  formatting:
    enabled: true
    tools: [{ name: php_cs_fixer }]
  type_checking:
    enabled: true
    tools: [{ name: phpstan }]
  security:
    enabled: true
    tools:
      - name: trivy
        domains: [sca]
      - name: opengrep
        domains: [sast]
  testing:
    enabled: true
    tools: [{ name: phpunit }]
  coverage:
    enabled: true
    threshold: 80
  duplication:
    enabled: true
    threshold: 5.0
fail_on:
  linting: error
  type_checking: error
  security: high
  testing: any
  coverage: below_threshold
  duplication: above_threshold
exclude:
  - "**/vendor/**"
  - "**/var/**"
  - "**/cache/**"
```

### Minimal (linting + security only)

```yaml
version: 1
project:
  languages: [php]
pipeline:
  linting:
    enabled: true
    tools: [{ name: phpcs }]
  type_checking:
    enabled: true
    tools: [{ name: phpstan }]
  security:
    enabled: true
    tools:
      - name: trivy
        domains: [sca]
      - name: opengrep
        domains: [sast]
exclude:
  - "**/vendor/**"
```
