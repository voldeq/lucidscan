# C#

**Support tier: Full**

C# projects are fully supported with linting, formatting, type checking, testing, coverage, security scanning, and duplication detection.

## Detection

| Method | Indicators |
|--------|-----------|
| **File extensions** | `.cs` |
| **Marker files** | `*.sln`, `*.csproj` (glob-based) |
| **Version detection** | `TargetFramework` from `.csproj`, `sdk.version` from `global.json` |

## Tools by Domain

| Domain | Tool | Notes |
|--------|------|-------|
| **Linting** | dotnet format (style) | Roslyn analyzer-based code style linting via `dotnet format style --verify-no-changes` |
| **Formatting** | dotnet format (whitespace) | Canonical .NET formatter; ships with .NET SDK |
| **Type Checking** | dotnet build | Roslyn compiler diagnostics (type errors, null reference warnings, CS* codes) |
| **Testing** | dotnet test | Supports xUnit, NUnit, MSTest; parses TRX reports |
| **Coverage** | dotnet coverage | Parses Cobertura XML from XPlat Code Coverage collector |
| **Security (SAST)** | OpenGrep | C#-specific vulnerability rules |
| **Security (SCA)** | Trivy | Scans `*.csproj` NuGet package references |
| **Duplication** | Duplo | Scans `.cs` files |

## Linting

**Tool: [dotnet format](https://learn.microsoft.com/en-us/dotnet/core/tools/dotnet-format)**

`dotnet format style` checks C# code against Roslyn analyzer style rules (IDE* diagnostics). Ships with the .NET SDK -- no separate installation required.

- Supports auto-fix via `dotnet format style`
- Configurable via `.editorconfig` or project-level analyzer settings
- Detects `.sln` or `.csproj` automatically (prefers `.sln`)
- Default rules: IDE0001--IDE0090 (naming, simplification, expression preferences)

```yaml
pipeline:
  linting:
    enabled: true
    tools:
      - name: dotnet_format
```

**Severity mapping:** Severity is determined by the diagnostic level:

- **High** -- `error` level diagnostics
- **Medium** -- `warning` level diagnostics
- **Low** -- `info` level diagnostics

**Installation:** Install the [.NET SDK](https://dotnet.microsoft.com/download) (dotnet format ships with it)

## Formatting

**Tool: [dotnet format whitespace](https://learn.microsoft.com/en-us/dotnet/core/tools/dotnet-format)**

`dotnet format whitespace` checks and fixes indentation, spacing, and line endings. Ships with the .NET SDK -- no separate installation required.

- Supports auto-fix (runs `dotnet format whitespace` directly)
- Check-only mode via `--verify-no-changes` (reports files that differ)
- Configurable via `.editorconfig`

```yaml
pipeline:
  formatting:
    enabled: true
    tools:
      - name: dotnet_format_whitespace
```

## Type Checking

**Tool: [dotnet build](https://learn.microsoft.com/en-us/dotnet/core/tools/dotnet-build)**

`dotnet build` runs the Roslyn C# compiler, which performs full type checking including nullable reference type analysis when enabled. Ships with the .NET SDK -- no separate installation required.

- Detects type errors, null reference issues, and compile-time diagnostics
- Supports strict mode (nullable reference types via `<Nullable>enable</Nullable>`)
- Parses MSBuild diagnostic format (`CS*` error codes)
- Operates on solutions or projects

**Severity mapping:** Severity is determined by the diagnostic code and level:

- **High** -- errors and known type-safety codes: CS0103, CS0246, CS8600, CS8602, CS8604
- **Medium** -- generic warnings

```yaml
pipeline:
  type_checking:
    enabled: true
    tools:
      - name: dotnet_build
```

## Testing

**Tool: [dotnet test](https://learn.microsoft.com/en-us/dotnet/core/tools/dotnet-test)**

`dotnet test` runs all unit tests using the configured test framework (xUnit, NUnit, or MSTest). Ships with the .NET SDK -- no separate installation required.

- TRX report output via `--logger trx` for detailed results
- Supports partial scanning (specific projects)
- Extracts failure messages and stack traces from TRX reports
- Falls back to console output parsing if TRX is unavailable

```yaml
pipeline:
  testing:
    enabled: true
    tools:
      - name: dotnet_test
```

## Coverage

**Tool: [XPlat Code Coverage](https://learn.microsoft.com/en-us/dotnet/core/testing/unit-testing-code-coverage)**

Parses Cobertura XML coverage reports generated by the XPlat Code Coverage collector via `dotnet test --collect:"XPlat Code Coverage"`.

- Parses existing Cobertura XML data produced by the test runner
- Returns error if no coverage report found (requires testing domain to be active)
- Per-file coverage with missing line tracking

> **Note:** When both the `testing` and `coverage` domains are active, the dotnet test runner automatically adds `--collect:"XPlat Code Coverage"` to generate coverage data in the same pass. No manual coverage generation is needed.

```yaml
pipeline:
  coverage:
    enabled: true
    tools:
      - name: dotnet_coverage
    threshold: 80
```

## Security

### SAST: OpenGrep

OpenGrep includes SAST rules for C# code, covering common .NET vulnerability patterns including SQL injection, XSS, insecure deserialization, and cryptographic weaknesses.

### SCA: Trivy

Trivy SCA scans NuGet package references in `.csproj` files for known vulnerabilities.

See the domain-specific sections in the [main documentation](../main.md) for details on OpenGrep, Trivy, and Checkov.

## Duplication

Duplo scans `.cs` files for duplicate code blocks.

```yaml
pipeline:
  duplication:
    enabled: true
    threshold: 5.0
```

## Timeouts

| Tool | Timeout | Rationale |
|------|---------|-----------|
| dotnet format (style) | 300s | Analyzes Roslyn diagnostics; large solutions need headroom |
| dotnet format (whitespace) | 120s | Pure formatting; fastest of the four |
| dotnet build | 300s | Full compilation; large solutions can be slow |
| dotnet test | 600s | Test suites can be inherently slow (integration tests, etc.) |

## Prerequisites

- **.NET SDK 6.0+** required (for `dotnet format` subcommands)
- **dotnet format**, **dotnet build**, **dotnet test** ship with the .NET SDK (no separate installation)
- Test framework NuGet packages (xUnit, NUnit, or MSTest) must be referenced in your `.csproj`

## Framework Detection

LucidShark auto-detects the following C# frameworks from `.csproj` package references:

| Framework | Detection |
|-----------|-----------|
| **ASP.NET Core** | `Microsoft.AspNetCore.*` packages or `Microsoft.NET.Sdk.Web` SDK |
| **Entity Framework** | `Microsoft.EntityFrameworkCore*` packages |
| **Blazor** | `Microsoft.AspNetCore.Components*` packages |
| **MAUI** | `Microsoft.Maui*` packages |
| **WPF** | `Microsoft.WindowsDesktop.App.WPF` |
| **WinForms** | `Microsoft.WindowsDesktop.App.WindowsForms` |

Test framework detection:

| Framework | Detection |
|-----------|-----------|
| **xUnit** | `xunit` package |
| **NUnit** | `NUnit` package |
| **MSTest** | `MSTest.TestFramework` package |
| **FluentAssertions** | `FluentAssertions` package |
| **Moq** | `Moq` package |

## Example Configuration

```yaml
version: 1
project:
  languages: [csharp]
pipeline:
  linting:
    enabled: true
    tools:
      - name: dotnet_format
  formatting:
    enabled: true
    tools:
      - name: dotnet_format_whitespace
  type_checking:
    enabled: true
    tools:
      - name: dotnet_build
  testing:
    enabled: true
    tools:
      - name: dotnet_test
  coverage:
    enabled: true
    tools:
      - name: dotnet_coverage
    threshold: 80
  security:
    enabled: true
    tools:
      - { name: trivy, domains: [sca] }
      - { name: opengrep, domains: [sast] }
  duplication:
    enabled: true
    threshold: 5.0
```
